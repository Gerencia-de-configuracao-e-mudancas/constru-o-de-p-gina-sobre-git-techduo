O que é?

O Git é um sistema de controle de versão distribuído amplamente utilizado para o gerenciamento de código-fonte e rastreamento de alterações em projetos de desenvolvimento de software. Ele foi criado por Linus Torvalds em 2005 e é uma ferramenta fundamental para desenvolvedores de software, pois ajuda a controlar e colaborar no desenvolvimento de projetos de software de maneira eficiente. Abaixo, estão alguns conceitos importantes relacionados ao Git:

Conceitos importantes



Por que usar o Git?

Fluxo de trabalho de ramificação de função

Uma das maiores vantagens do Git são seus funções de branch essas funções facilitam o fluxo de trabalho da ramificação de funções popular entre muitos usuários do Git. Os branches de características oferecem um ambiente isolado para cada alteração na base de código.

Desenvolvimento distribuído

O Git é um sistema de controle de versão distribuído. Em vez de uma cópia de trabalho, cada desenvolvedor obtém seu próprio repositório local, com um histórico completo de commits. Semelhante aos branches de característica, o desenvolvimento distribuído cria um ambiente mais confiável.

Solicitações pull

A pull request é uma forma de pedir a outro desenvolvedor para fazer o merge de um dos seus branches no repositório dele. Além de facilitar o acompanhamento das alterações para os líderes de projeto, esse recurso também possibilita que os desenvolvedores dialoguem antes de integrar o trabalho à base de código.

Git Commands

git init

Inicializa um repositório Git e diretório local, criando uma estrutura Git vazia para rastrear e versionar arquivos.

git clone

Cria uma cópia de um repositório Git existente. A clonagem é a forma mais comum de os desenvolvedores obterem uma cópia de trabalho de um repositório central.

git add

Move as alterações do diretório de trabalho para a área de teste. Isso lhe dá a oportunidade de preparar um instantâneo antes de enviá-lo para a história oficial.

git commit

Registra as alterações e confirma no histórico do projeto. Combinado com git add, isso define o fluxo de trabalho básico para todos os usuários do Git.

git pull

Ele baixa um branch de um repositório remoto e imediatamente o mescla no branch atual.

git push

Ele permite mover uma ramificação local para outro repositório, o que serve como uma maneira conveniente de publicar contribuições.

git branch

Ele permite criar ambientes de desenvolvimento isolados em um único repositório. Também pode ser usado para listar, criar e excluir branches.

git merge

Serve para integrar mudanças de uma branch em outra.

Fluxo de Trabalho com Git

Iniciar um Projeto

Use 'git init' para criar um repósitorio no diretório local ou use 'git clone' para clonar um repositório existente

Fazer mudanças

Edite e crie arquivos e use o 'git add' para prepará-los para commit

Commit das mudanças

Use 'git commit' para criar um registro das mudanças

Trabalho em equipe

Use branches para trabalhar em paralelo com outros colegas e o 'git merge' para unir as alterações

Atualização e Envio

Use 'git pull' para obter as mudanças remotas e 'git push' para mandar suas alterações para o repósitorio renoto
