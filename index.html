O que é?

O Git é um sistema de controle de versão distribuído amplamente utilizado para o gerenciamento de código-fonte e rastreamento de alterações em projetos de desenvolvimento de software. Ele foi criado por Linus Torvalds em 2005 e é uma ferramenta fundamental para desenvolvedores de software, pois ajuda a controlar e colaborar no desenvolvimento de projetos de software de maneira eficiente. Abaixo, estão alguns conceitos importantes relacionados ao Git:

Git Commands

git init

Inicializa um repositório Git e diretório local, criando uma estrutura Git vazia para rastrear e versionar arquivos.

git clone

Cria uma cópia de um repositório Git existente. A clonagem é a forma mais comum de os desenvolvedores obterem uma cópia de trabalho de um repositório central.

git add

Move as alterações do diretório de trabalho para a área de teste. Isso lhe dá a oportunidade de preparar um instantâneo antes de enviá-lo para a história oficial.

git commit

Registra as alterações e confirma no histórico do projeto. Combinado com git add, isso define o fluxo de trabalho básico para todos os usuários do Git.

git pull

Ele baixa um branch de um repositório remoto e imediatamente o mescla no branch atual.

git push

Ele permite mover uma ramificação local para outro repositório, o que serve como uma maneira conveniente de publicar contribuições.

git branch

Ele permite criar ambientes de desenvolvimento isolados em um único repositório. Também pode ser usado para listar, criar e excluir branches.

git merge

Serve para integrar mudanças de uma branch em outra.
